# BOSS直聘岗位信息爬虫 — 使用说明

## 第1步：安装依赖

确保在 `crawler/` 目录下：
```bash
npm install
```

## 第2步：安装 Chrome 扩展

1. 打开 Chrome → `chrome://extensions`
2. 开启右上角的 **"开发者模式"**
3. 点击左上角 **"加载已解压的扩展"**
4. 选择 `chrome-extension` 目录

> 扩展的作用：从浏览器请求中自动捕获 cookies、zp_token、token 等认证信息，并发送给爬虫程序。
>
> **注意：扩展会每 10 秒自动刷新一次 zhipin.com 页面来保持 token 新鲜，这是正常的。**



## 第3步：启动后端

在终端运行：

```bash
node src/scraper.js
```



## 第4步：登录 BOSS直聘，并在插件指引下操作

在同一个 Chrome 浏览器中打开 [Boss直聘](https://www.zhipin.com) 并登录。此时，可以看到右下角有对应的UI界面指引操作。

![image-20260619000637157](https://raw.githubusercontent.com/Just-A-Freshman/image-bed/main/Typora/image-20260619000637157.png)

此时确保Boss处于【**职位**】这个页面，然后点击右手边的“+”，新建一个项目，输入项目名称。

![image-20260619000834006](https://raw.githubusercontent.com/Just-A-Freshman/image-bed/main/Typora/image-20260619000834006.png)

关键点来了：怎么筛选岗位呢？你必须**先在输入框内搜索一个关键词然后点击搜索**，然后选择下面的各项信息：城市、求职信息、薪资待遇、工作经验等等。这个顺序不能反：即必须先搜索，再选细节信息。

选完以后，点击“重新捕获”，你可以看到关键词处发生了变化。然后，点击创建就可以开始爬取了！

Q1：显示“**连接爬虫失败**”，一定是因为你没有启动后端（第三步）！

Q2：登录boss直聘后，马上**插件会新建一个boss直聘窗口**，并在另外一个窗口不断刷新。这是第二步提到的插件行为，是正常的Token保活行为。不要关闭那个窗口！！



## 其他设置项

`data/settings.json`有完整的设置选项，不推荐自己编辑`apiParams`相关的信息。我们只着重说明下面的几个参数：

```json
"maxPages": 30,                # 单次爬取的最大页数
"delayBetweenPages": 6000,     # 获取总爬取列表时的翻页时间间隔，默认为6秒
"delayBetweenDetails": 1000,   # 获取详情页的时间间隔，默认为1秒
"retryMax": 3,                 # 最大重试次数
"retryDelay": 5000             # 重试时间间隔，默认为5秒
```

这自然引申出一个问题：

- 如果你想爬取得更快，可以尝试调小`delayBetweenDetails`，但这需要更高的页面刷新速度，而且有被封IP的风险；
- 如果你想单次爬取更多的岗位，那么就调大`maxPages`。

项目显然不能像哪些真正的并发爬虫那么快，能并发地进行爬虫一定是做了JS逆向分析的，其工作量不仅大而且难，而且对面一更新，你就又需要重新分析。作者实在是菜得抠脚，做不到这一点。

因此我必须承认，这个爬虫项目的目的本身不是为了大规模爬虫设计的，恰恰相反，它的目的是为了把你附近的心仪岗位全部聚合在一起，方便AI的快速分析。我相信在你居住位置的附近，满足基本要求的岗位数量不可能真的能达到数千。更常见的情况是，你只需要在几百个岗位中作抉择，针对他们修改简历。这正是本爬虫项目希望做到的。
